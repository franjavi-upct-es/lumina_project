    PPO Agent and Uncertainty Gate¶

    Relevant source files
    • backend/cognition/agent/ppo_agent.py
    • backend/cognition/training/behavioral_cloning.py
    • backend/config/constants.py
    • backend/simulation/arena/schemas.py
    • notebooks/06_uncertainty_calibration.ipynb

    The Cognition Layer (Layer 3) of the Lumina V3 architecture is responsible for high-level decision-making. It consumes the 256-d latent state from the Deep Fusion Nexus and maps it to a 4-dimensional continuous action space. This layer implements a Proximal Policy Optimization (PPO) agent equipped with an Uncertainty Gate, a safety mechanism that monitors epistemic uncertainty via MC-Dropout to prevent the agent from acting in regimes where the model is "confused".

    PPO Agent Implementation¶

    The PPOAgent class manages the policy network, the rollout buffer, and the update logic. It follows the clipped surrogate objective approach to ensure stable policy updates backend/cognition/agent/ppo_agent.py#135-152

    Action Space¶

    The agent operates in a 4-D squashed-Gaussian continuous action space backend/config/constants.py#26-33:

    • Action[0] (Direction): Range [-1, 1], representing full short to full long.
    • Action[1] (Urgency): Range [-1, 1], representing limit/maker to market/taker.
    • Action[2] (Sizing): Range [-1, 1], representing the fraction of risk capital (Kelly-fraction multiplier).
    • Action[3] (Stop Distance): Range [-1, 1], representing tight to wide stops in ATR multiples.

    Core Objectives and Loss¶

    The agent optimizes a total function \(L\) composed of three terms backend/cognition/agent/ppo_agent.py#17-22:

    1. Clipped Surrogate Objective (\(L^{CLIP}\)): Limits the policy update step size to prevent catastrophic performance collapse backend/cognition/agent/ppo_agent.py#6-9
    2. Value Loss (\(L^{VF}\)): A clipped MSE loss between predicted values and GAE-calculated returns backend/cognition/agent/ppo_agent.py#21-22
    3. Entropy Bonus (\(H\)): Encourages exploration by penalizing overly deterministic policies backend/cognition/agent/ppo_agent.py#72-73

    Advantage Estimation¶

    The agent utilizes Generalized Advantage Estimation (GAE) to balance bias and variance in the advantage signal backend/cognition/agent/ppo_agent.py#11-16

    • \(\gamma\) (Discount Factor): Set to 0.99 for an effective horizon of ~100 steps backend/cognition/agent/ppo_agent.py#59-61
    • \(\lambda\) (GAE Lambda): Set to 0.95 to provide a smooth trade-off between TD(0) and Monte Carlo returns backend/cognition/agent/ppo_agent.py#62-64

    Training Flow and Early-Stop¶

    During the update phase, the agent iterates over the RolloutBuffer for a fixed number of epochs (default 10) backend/cognition/agent/ppo_agent.py#76-77 It monitors KL divergence between the old and new policy; if the divergence exceeds \(1.5 \times \text{target_k}\) (default 0.2), the update is terminated early to prevent policy drift backend/cognition/agent/ppo_agent.py#82-84

    Sources: backend/cognition/agent/ppo_agent.py#1-164 backend/config/constants.py#24-66

    Uncertainty Gate¶

    The UncertaintyGate is a "hysteresis-based" safety mechanism that sits outside the core RL loop. It evaluates the model's confidence in its own predictions using Monte-Carlo (MC) Dropout backend/cognition/agent/ppo_agent.py#27-31

    MC-Dropout Mechanism¶

    When the agent's act() method is called, it performs \(N=10\) forward passes through the PolicyNetwork with dropout enabled backend/cognition/agent/ppo_agent.py#186-191

    1. Sampling: \(N\) action vectors are sampled.
    2. Variance Calculation: The variance across these samples represents the epistemic uncertainty notebooks/06_uncertainty_calibration.ipynb
    3. Normalization: The variance is normalized to a \([0, 1]\) scalar notebooks/06_uncertainty_calibration.ipynb

    Gate Logic: Hysteresis and Veto¶

    The gate uses two thresholds, \(\tau_{high}\) and \(\tau_{low}\), to manage state transitions:

    • Engagement: If uncertainty exceeds \(\tau_{high}\), the gate engages and replaces the agent's action with a Defensive Action (usually "Neutral/Cash").
    • Disengagement: The gate only disengages when uncertainty falls bellow \(\tau_{low}\). This prevents "flickering" in high-volatility regimes notebooks/06_uncertainty_calibration.ipynb
    • Critical Veto: If the gate vetoes 50 consecutive steps, it triggers a CRITICAL log entry, suggesting a potential data-feed failure or major distributional shift notebooks/06_uncertainty_calibration.ipynb

    Calibration (Threshold Sweep)¶

    Thresholds are picked using a precision/recall trade-off. The goal is to maximize Adversarial Recall (catching 90% of adversarial regimes) while maintaining Coverage (allowing the agent to act in 80% of healthy regimes) notebooks/06_uncertainty_calibration.ipynb

    Sources: backend/cognition/agent/ppo_agent.py notebooks/06_uncertainty_calibration.ipynb

    Behavioral Cloning (Phase A)¶

    Before PPO training (Phase B), the PolicyNetwork is warm-started using Behavioral Cloning (BC). This prevents the agent from starting with random noise and ensures it begins in a region of parameter space that produces coherent actions backend/cognition/training/behavioral_cloning.py#2-8

    Loss Function¶

    BC minimizes the Negative Log-Likelihood (NLL) of expert actions under the current policy backend/cognition/training/behavioral_cloning.py#12-16:

    \[ L(\theta)=-\mathbf{E}_{(s,\alpha^*)\sim D}[\log\pi_\theta(a^*|s)] \]

    This is preferred over MSE because it avoids tanh saturation at the action boundaries (\(\pm1\)) and effectively calibrates the variance (log-std) of the policy for future PPO exploration backend/cognition/training/behavioral_cloning.py#20-29

    Validation Metric¶

    The Spartan Curriculum uses a bc_min_accuracy gate (default 0.55). Accuracy is defined as the fraction of validation samples where the policy's deterministic action has the same sign-pattern (directionality) as the expert across all 4 dimensions backend/cognition/training/behavioral_cloning.py#39-42
