    Temporal Encoder: Temporal Fusion Transformer (TFT)¶

    Relevant source files
    • backend/cognition/training/behavioral_cloning.py
    • backend/config/constants.py
    • backend/perception/temporal/inference.py
    • notebooks/02_tft_prototype.ipynb

    The Temporal Encoder is the first component of the Perception Layer in the Chimera architecture. It is responsible for transforming raw multi-variate price data (OHLCV) into a high-dimensional Temporal Embedding (\(d=128\)). It utilizes a custom implementation of the Temporal Fusion Transformer (TFT) (Lim et al., 2021), optimized for financial time-series by incorporating gated residual networks and variable selection to handle noisy market data.

    1. Architecture Overview¶

    The TemporalFusionTransformer class backend/perception/temporal/tft_model.py#60-72 implements a multi-stage pipeline designed to extract both long-term dependencies and local patterns from market bars.

    Key Components¶

    • Gated Residual Network (GRN): The fundamental building block backend/perception/temporal/tft_model.py#28-32 It provides adaptive non-linear processing with a GLU (Gated Linear Unit) to skip unnecessary steps, preventing vanishing gradients in deep financial networks.
    • Variable Selection Network (VSN): Performs instance-specific feature selection backend/perception/temporal/tft_model.py#42-57 It assign weights to different OHLCV inputs, allowing the model to ignore features (e.g., Volume) if they are currently non-informative.
    • LSTM Encoder: Processes the sequence of variable-selected features to capture local temporal context backend/perception/temporal/tft_model.py#85-91
    • Interpretable Multi-Head Attention: A specialized attention mechanism that allows the model to focus on specific past events (e.g., a flash crash 30 minutes ago) backend/perception/temporal/tft_model.py#93-98
    • Temporal Embedding Output: The final state is projected to a 128-dimensional vector defined by DIM_PRICE backend/config/constants.py#56-57

    2. Implementation Details¶

    Variable Selection Network (VSN)¶

    The VSN is critical for financial data where the signal-to-noise ratio varies. It computes a vector of weights \(\alpha\) for the input features and produces a weighted sum of transformed features.

    • Function: VariableSelectionNetwork.forward backend/perception/temporal/tft_model.py#50-57
    • Mechanism: It uses a GRN to produce weights for each input variable and another set of GRNs to transform each individual variable before weighting them.

    Gated Residual Network (GRN)¶

    The GRN allows the model to adaptively control the flow of information.

    • Function: GatedResidualNetwork.forward backend/perception/temporal/tft_model.py#35-39
    • Logic: x -> Linear -> ELU -> Linear -> GLU -> AddNorm.

    Dimensional Constants¶

    The TFT's output is strictly governed by project-wide constants to ensure compatibility with the DeepFusionNexus.

    Constant Value Description
    DIM_PRICE 128 Size of the temporal embedding vector.
    OHLCV_WINDOW_MINUTES 240 Look-back window (4 hours of 1-min bars).
    OHLCV_HORIZON_MINUTES 60 Target horizon for self-supervised training.

    Sources: backend/config/constants.py#56-95 backend/perception/temporal/tft_model.py#28-140

    3. Training and Preprocessing¶

    Self-Supervised Pre-training¶

    The TFT is pre-trained using a self-supervised task: predicting the forward return over OHLCV_HORIZON_MINUTES backend/config/constants.py#95 This ensures the encoder learns to extract features relevant to price movement before being integrated into the RL agent's policy.

    Preprocessing Pipeline¶

    1. Normalization: The TFTInferenceService applies Min-Max scaling to the input buffer backend/perception/temporal/inference.py#68
    2. Windowing: A sliding window of 240 bars is maintained in a deque backend/perception/temporal/inference.py#33
    3. Outlier Handling: The prototype utilizes RobustScaler to mitigate the impact of price spikes notebooks/02_tft_prototype.ipynb

    Sources: backend/perception/temporal/inference.py#31-62 notebooks/02_tft_prototype.ipynb

    4. Inference Service¶

    The TFTInferenceService handles real-time embedding generation. It subscribes to a Redis tick stream and writes embeddings back to the Feature Store.
